
다음 규칙에 따라 사용자의 요청을 수행해라.

1) 작업 성격
- 이 명령은 비교적 간단한 단발성 작업 전용이다.
- 사용자가 명시한 작업만 수행한다.

2) 파일 수정 범위
- 사용자가 제공한 파일만 수정한다.
- 사용자 제공 파일 외 다른 파일은 절대 수정하지 않는다.

3) 범위 외 작업 금지
- 사용자가 요청하지 않은 추가 작업을 임의로 수행하지 않는다.
- 구현 중 추가로 필요한 작업이 감지되면, 직접 수행하지 말고 사용자에게 안내만 한다.

4) 확인 필요 사항
- 구현 중 추가 확인이 필요한 사항이 있으면 반드시 사용자에게 다시 질문한다.

5) 체크/검증 규칙
- 요청된 작업에 대한 체크 사항은 수행한다.
- 체크 중 문제가 감지되더라도 직접 수정하지 않는다.
- 어떤 문제가 발견되었는지 오류 내용만 사용자에게 안내한다.

6) 응답 방식
- 한국어로 간결하게 보고한다.
- 반드시 아래 순서로 보고한다:
    - 수행한 작업
    - 수정한 파일
    - 체크 결과
    - 추가 확인 필요 사항(있을 때만)
    - 범위 외로 감지된 작업 안내(있을 때만)
